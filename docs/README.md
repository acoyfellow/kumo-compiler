# Kumo documentation

This documentation is organized into four sections, each serving a different
need:

- **[Tutorials](./tutorials)** — learning-oriented lessons. Start here if you
  are new. They take you from nothing to a working component.
- **[How-to guides](./how-to)** — task-oriented recipes for getting a specific
  job done (deploy, add a component, drop Kumo into an existing app).
- **[Reference](./reference)** — information-oriented. The full component list
  and the package APIs.
- **[Explanation](./explanation)** — understanding-oriented background on how
  the compiler works and why it emits native code instead of wrappers.

If you don't know where to start, read
[tutorials/getting-started](./tutorials/getting-started.md).
