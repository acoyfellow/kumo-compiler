# The Facet-Compiler Trajectory — 10 probes out

Each probe is designed to BREAK the previous happy result. That is the point. A
research program that only confirms is theater (we lived that). Every step states:
the question, what would falsify the thesis, the cheapest experiment, and what we
learn either way. Grounded in what is already proven.

Status legend: [proven] [next] [planned] [speculative] [probably-a-trap-if-skipped]

---

## Where we stand (the base camp — all proven)
- WEAK: a Worker is a compiler pass. [proven]
- MEDIUM: the edge is the IR substrate; memoized (cold 459ms → warm 37ms). [proven]
- STRONG: a model-generated artifact passes a mutation-tested interaction gate. [proven]
- ECON: ~100% first-pass @ ~1.5k tokens / ~33s per verified component; correctness
  from the gate, not determinism (9/10 distinct outputs, all passed). [proven, one component]
- THE LEVER (hypothesis to attack): first-pass rate is a function of BRIEF (Contract
  IR) quality, not model power.

---

## Probe 1 — Break first-pass with a HARDER component  [next]
**Question:** does ~100% survive command-palette (combobox-in-dialog) or date-picker
(calendar grid), where trace-reconstruction and the naive port both struggled?
**Falsifies if:** first-pass collapses (<50%) even with a good brief → model power,
not brief quality, is the ceiling.
**Experiment:** same protocol, swap the component + Contract IR. N=10.
**Learn either way:** where the model's competence boundary actually is, and whether
the repair path finally earns its keep (it was built, never exercised).

## Probe 2 — Break the brief: how little can the Contract IR say?  [planned]
**Question:** if THE LEVER is real, degrading the brief should degrade first-pass
monotonically. Run the SAME component at 4 brief tiers: full → obligations-only →
one-line → just "port this React component."
**Falsifies if:** first-pass is flat across tiers → the model doesn't need the brief,
and our hard-won lessons are not the value (uncomfortable but important to know).
**Learn:** the dose-response curve of specification → the actual unit of value is
quantified. This is the experiment that proves or kills "brief quality is the lever."

## Probe 3 — Break the gate's honesty at scale (adversarial mutations)  [planned]
**Question:** the gate rejected 2 hand-made mutations. Does it survive 50 diverse
ones — including SUBTLE theater (opens but focus never moves; opens but Escape
doesn't close; opens visually but role=menu missing)?
**Falsifies if:** any plausible-but-wrong artifact passes → the oracle is too weak;
fix the oracle, not the facet (the cardinal rule).
**Experiment:** a mutation corpus (generated by a SECOND model asked to "produce
convincing-but-broken variants") fed through the gate; require 100% rejection.
**Learn:** the gate's true discriminative power. This is the most important defensive
probe — a weak oracle invalidates every green above it.

## Probe 4 — Break determinism's absence: pin reproducibility via receipts  [planned]
**Question:** outputs are non-deterministic (9/10 distinct). Can content-addressed
receipts + pinned model + frozen brief make a COMPILE reproducible enough to trust
and cache, even though the artifact varies?
**Falsifies if:** identical inputs yield gate-passing outputs with materially
different BEHAVIOR (not just different text) → "passes the gate" under-specifies
behavior and the IR/oracle must tighten.
**Experiment:** differential-test the 9 distinct passing outputs against each other
AND against canonical React: do they behave identically, or merely each-pass?
**Learn:** whether the gate defines behavior tightly or admits an equivalence class
that is too loose. Directly tests the e-graph "admission vs ranking" distinction.

## Probe 5 — Break single-model: speculative ensemble fan-out  [planned]
**Question:** the egraphs pillar predicted speculative fan-out. Run N models /
N prompts per component in parallel (spawn_batch race/quorum), keep first gate-pass.
Does plurality raise final-pass on HARD components (Probe 1's failures)?
**Falsifies if:** ensemble adds cost without raising pass rate on hard cases → the
failures are spec failures, not sampling failures (re-prompting won't help).
**Experiment:** for components that failed Probe 1, race 3 candidates; measure
pass-rate lift vs token cost.
**Learn:** when speculation pays (sampling-bound failures) vs when it is waste
(spec-bound failures). The economic decision rule for fan-out width.

## Probe 6 — Break the local gate: run verification AT THE EDGE  [planned]
**Question:** today the gate runs locally (Vite + headless Chrome). Can the type
checker itself live on Cloudflare — Browser Rendering for the interaction, a Worker
orchestrating? Then the WHOLE compiler (generate + verify) is edge-native.
**Falsifies if:** edge browser rendering can't do trusted pointer input / DOM
assertions, or latency/cost makes per-request verification infeasible → verification
stays an offline build step, weakening "compile on the request path."
**Experiment:** port gate.mjs to a Worker + Browser Rendering binding; re-run Probe
1's component end-to-end on the edge.
**Learn:** whether "the isolate is the type checker" is literally true in prod, or a
local-only convenience. This is the rung that makes the thesis fully edge-resident.

## Probe 7 — Break ported-once: the INCREMENTAL/UPSTREAM compile  [planned]
**Question:** the original project premise was Kumo CHANGES upstream. When canonical
React bumps 2.5.2 → 2.6.0, can the facet compiler detect the diff, re-synthesize only
affected components, and re-gate — content-addressed so unchanged components are
cache hits (the medium-rung memoization, applied to source change)?
**Falsifies if:** small upstream diffs cause broad cache invalidation / full
re-synthesis → the IR doesn't isolate change well; incrementality is fake.
**Experiment:** simulate an upstream bump (mutate one canonical component), run the
compiler, measure how many facets re-fire vs cache-hit.
**Learn:** whether this is a one-shot porter or a living compiler. This is where a
compiler becomes justified over a naive loop (the retro's "500 components weekly").

## Probe 8 — Break the human-out-of-loop assumption: escalation discipline  [planned]
**Question:** when a component genuinely cannot pass (Probe 1 hard failures after
repair + ensemble), does the system SURFACE a novel blocker cleanly to a human
instead of thrashing or faking green? Test the escalation path, not just success.
**Falsifies if:** the system loops/retries forever, or worse, relaxes the gate to
declare victory → the self-modifying-gate trap (a flagged danger).
**Experiment:** feed an impossible obligation (a component with no Ark primitive);
verify it escalates with a precise blocker after bounded attempts, gate untouched.
**Learn:** the repair→escalate boundary; proves the system fails safe, not silently.

## Probe 9 — Break the toy scale: full 41-component fan-out on the edge  [speculative]
**Question:** everything above is 1 component. Run the entire Kumo set × 3 frameworks
(≈123 facets) as one edge fan-out with the per-component economics from Probe 2 and
the ensemble policy from Probe 5. Does it hold, and what does it cost?
**Falsifies if:** DO facet limits, account quotas, or aggregate cost make planet-scale
fan-out infeasible → the architecture is elegant but doesn't scale economically.
**Experiment:** the prod Worker, but Compiler.compileAll() calls synthesize+gate for
all components; collect the full ledger (cost, pass-rate distribution, tail latency).
**Learn:** the real cost curve and failure distribution across the WHOLE surface —
the number that decides whether this beats a human writing the passes.

## Probe 10 — Break the compiler itself: mine deterministic passes from the corpus  [speculative]
**Question:** the findings doc's deepest move — after N verified compiles, do the
model's outputs CONVERGE on repeated mappings (e.g. "Kumo Dialog → Ark Dialog with
these exact classes")? If so, distill those into a CHEAP DETERMINISTIC pass and
reserve the expensive facet for the residual. The compiler compiles itself toward
classical.
**Falsifies if:** outputs never stabilize (every compile is novel) → no extractable
rules; it stays an expensive agent loop forever (re-examine whether a compiler is
even the right tool vs the naive loop — the retro's open question).
**Experiment:** cluster the accepted outputs from Probes 1-9 by structure; where a
mapping repeats ≥K times with gate-pass, generate a deterministic emitter for it;
measure how much of the surface becomes free.
**Learn:** whether "compile by example" closes the loop — expensive exploration
amortized into cheap deterministic passes. This is the endgame: the agent-native
compiler earning the right to become a normal compiler where it can, and staying
agentic only where it must.

## The shape of the trajectory (read this)

```
Probe 1-2:  WHERE does it break, and is the brief really the lever?   (competence + dose-response)
Probe 3-4:  is the GATE honest, and does "passes" mean "correct"?      (oracle integrity)
Probe 5-6:  speculation + edge-native verification                     (the architecture, fully)
Probe 7-8:  incremental + fail-safe                                    (a LIVING, safe compiler)
Probe 9:    does it scale economically across the whole surface?       (the go/no-go number)
Probe 10:   does it amortize into cheap deterministic passes?          (the endgame)
```

The through-line: **stop trying to confirm the thesis; spend every probe trying to
break it.** The thesis is only as strong as the hardest break it survives. Probe 3
(gate honesty) and Probe 7 (incrementality) are the two that, if they fail, sink it —
prioritize them. Probe 2 (dose-response) is the cheapest highest-information shot and
should run first alongside Probe 1.

## The one-sentence horizon
If all 10 hold: an edge-resident, self-verifying, incrementally-recompiling compiler
whose stochastic passes are amortized into deterministic ones over time — i.e.
"Cloudflare as a compiler" not as a metaphor but as a running, economical system.
If Probe 3 or 7 fails: a very good one-shot porter with an honest gate — still
valuable, but not a compiler. Either outcome is worth knowing precisely.
