Evidence and readiness

The package version is intentionally boring. Readiness comes from what was executed against the packed artifact.

Package proof

Each library receipt covers deterministic packing, file inventory, installation, type checking, client build, SSR build/render, hydration, server-node preservation, Button behavior, Field model and ARIA, CSS assets, tree-shaking, and workspace use.

Browser proof

The production galleries run the installed tarballs at mobile, tablet, and desktop widths. They exercise click behavior, disabled invariants, model updates, label/description/error/required ARIA, CSS, same-origin framing, and network/console health.

What remains not-run

  • Hot-module replacement as a package gate
  • Manual screen-reader sessions

Those checks remain explicitly not-run; they are not converted into passes.

Why Select is absent

Select has contract and structural evidence, but its framework browser and consumer gates are incomplete or failed. The packages therefore expose Button and Field only. See the Select pilot.

Separate compiler evidence

Planner benchmarks, output-architecture experiments, the immutable 164/164 runtime authority, and package readiness are different claims. A fast planner does not choose an architecture, and a generated source file is not a browser pass.
