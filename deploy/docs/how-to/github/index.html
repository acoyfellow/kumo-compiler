How-to guide

Install from one GitHub repository

The monorepo publishes the exact proven npm tarballs as assets on one GitHub Release. No separate framework repositories are required.

Install a release asset

npm install https://github.com/<owner>/kumo-compiler/releases/download/libraries-v0.0.1/kumo-vue-0.0.1.tgz
npm install https://github.com/<owner>/kumo-compiler/releases/download/libraries-v0.0.1/kumo-svelte-0.0.1.tgz
npm install https://github.com/<owner>/kumo-compiler/releases/download/libraries-v0.0.1/kumo-solid-0.0.1.tgz

Replace <owner> after the repository remote exists. The release includes SHA256SUMS and a machine-readable manifest.

Why a release asset?

npm Git dependencies install the package at the repository root. This monorepo contains three package roots below dx/packages/, so github:owner/kumo-compiler#tag would install the compiler repository—not one framework package. Release tarballs keep one repository and preserve the exact packed bytes already tested.

Prepare the release

npm run libraries:build
KUMO_GITHUB_OWNER=<owner> KUMO_GITHUB_REPO=kumo-compiler npm run release:github
npm run release:github:verify

The generated bundle lives at release/github/libraries-v0.0.1/. GitHub Actions repeats this preparation and uploads it as a workflow artifact; creating the public GitHub Release remains an authorized human action.
