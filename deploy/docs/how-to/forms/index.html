How-to guide

Build a form with Field

Give every Field a stable ID, a visible label, and the state that belongs on its native input.

Provide an ID and label

<Field id="email" label="Email address" bind:value={email} />

The label’s for points to the generated native input.

Add description and error text

<Field
  id="email"
  label="Email address"
  description="We only use this for account notices."
  error={email.includes('@') ? undefined : 'Enter a valid email.'}
  required
  bind:value={email}
/>

Field creates email-description and email-error, joins them in aria-describedby, sets aria-invalid="true" while an error exists, and gives the error role="alert".

Use native form behavior

required, disabled, name, autocomplete, input type, and other native attributes are forwarded. Use the framework’s normal model or input event:

  • Svelte: bind:value
  • Vue: v-model
  • Solid: value plus onInput
Field does not own validation policy or form submission. It renders state and accessibility relationships supplied by your application.
