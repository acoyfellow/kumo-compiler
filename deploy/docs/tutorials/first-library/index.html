Tutorial

Your first Kumo library

Build the same small interaction in Vue, Svelte, or Solid. This tutorial uses the exact production artifacts exercised by the package receipts.

1 — Choose a package

# Vue
npm install https://kumo-compiler.coey.dev/packages/kumo-vue-0.0.1.tgz

# Svelte
npm install https://kumo-compiler.coey.dev/packages/kumo-svelte-0.0.1.tgz

# Solid
npm install https://kumo-compiler.coey.dev/packages/kumo-solid-0.0.1.tgz

Use only one command in your project. npm installs the matching framework peer dependency if your app does not already provide it.

2 — Import the library and its styles

Svelte

<script>
  import { Button, Field } from '@cloudflare/kumo-svelte';
  import '@cloudflare/kumo-svelte/styles.css';

  let clicks = $state(0);
  let project = $state('Workers');
</script>

<Button onclick={() => clicks += 1}>Clicked {clicks}</Button>
<Field
  id="project"
  label="Project"
  description="A production project name."
  required
  bind:value={project}
/>

<p>Model: {project}</p>

Vue

<script setup>
import { ref } from 'vue';
import { Button, Field } from '@cloudflare/kumo-vue';
import '@cloudflare/kumo-vue/styles.css';

const clicks = ref(0);
const project = ref('Workers');
</script>

<template>
  <Button @click="clicks++">Clicked {{ clicks }}</Button>
  <Field id="project" label="Project" description="A production project name." required v-model="project" />
  <p>Model: {{ project }}</p>
</template>

Solid

import { createSignal } from 'solid-js';
import { Button, Field } from '@cloudflare/kumo-solid';
import '@cloudflare/kumo-solid/styles.css';

export default function App() {
  const [clicks, setClicks] = createSignal(0);
  const [project, setProject] = createSignal('Workers');
  return <>
    <Button onClick={() => setClicks(value => value + 1)}>Clicked {clicks()}</Button>
    <Field id="project" label="Project" description="A production project name." required value={project()} onInput={event => setProject(event.currentTarget.value)} />
    <p>Model: {project()}</p>
  </>;
}

3 — Check the contract

  • The button starts at Clicked 0 and increments.
  • The label targets the input ID.
  • Description and error IDs are composed into aria-describedby.
  • required, disabled, and aria-invalid reach native controls.
  • The package stylesheet supplies visible focus, state, spacing, and minimum 44px control height.
