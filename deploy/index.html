Kumo compiler study

Which implementation should compile Kumo?

We have one working TypeScript reference compiler and completed protocol planner benchmarks in Go, Rust, and Zig. The native planners prove the shared contract and deterministic planning path; the full emitter bake-off is next before any reference implementation is chosen.

One benchmark contract

Every candidate must consume the same kumo.ir/v1 input, produce equivalent framework source, pass the same browser and provenance gates, and report raw repeatable measurements. A faster compiler does not win if its output or developer workflow is weaker.
